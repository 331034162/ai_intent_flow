# 会议室预订工具业务规则文档

## 一、功能概述

提供会议室查询和预订功能。

---

## 二、查询会议室可用时间

### 2.1 时间参数规则

| 用户输入 | 系统处理 |
|----------|----------|
| 无参数 | 当前时间 < 20点：查今天、明天、后天（共3天）<br>当前时间 ≥ 20点：查明天、后天（共2天） |
| "今天" | 当前时间 至 今天 20:00 |
| "明天" / "后天" / 具体日期 | 该日期 08:00 - 20:00 |
| 有时间段无日期（如"下午3点到5点"） | 查今天的指定时间段 |
| 跨天查询（如"今天到后天"） | 支持最多 10 天跨度 |
| 最近N天（如"最近三天"、"最近七天"） | 查当前时间往后N天的范围，N最大为10 |

### 2.2 用户输入示例

#### 场景一：无时间参数

| 用户输入 | 当前时间 | 查询范围 |
|----------|----------|----------|
| "查会议室" | 2026-04-14 10:00:00 | 2026-04-14 10:00:00 至 2026-04-16 20:00:00（共3天） |
| "会议室情况" | 2026-04-14 21:00:00 | 2026-04-15 08:00:00 至 2026-04-16 20:00:00（共2天） |

#### 场景二：仅日期

| 用户输入 | 当前时间 | 查询范围 |
|----------|----------|----------|
| "查今天的会议室" | 2026-04-14 10:00:00 | 2026-04-14 10:00:00 至 2026-04-14 20:00:00 |
| "查今天的会议室" | 2026-04-14 21:00:00 | 报错：查询的时间段不能早于当前时间 |
| "明天有什么会议室" | 2026-04-14 10:00:00 | 2026-04-15 08:00:00 至 2026-04-15 20:00:00 |
| "后天有空闲的会议室吗" | 2026-04-14 10:00:00 | 2026-04-16 08:00:00 至 2026-04-16 20:00:00 |
| "5月1号的会议室" | 2026-04-14 10:00:00 | 2026-05-01 08:00:00 至 2026-05-01 20:00:00 |

#### 场景三：仅时间段（默认今天）

| 用户输入 | 当前时间 | 查询范围 |
|----------|----------|----------|
| "下午3点到5点" | 2026-04-14 10:00:00 | 2026-04-14 15:00:00 至 2026-04-14 17:00:00 |
| "上午9点到11点" | 2026-04-14 10:00:00 | 2026-04-14 09:00:00 至 2026-04-14 11:00:00 |
| "晚上7点到8点" | 2026-04-14 10:00:00 | 2026-04-14 19:00:00 至 2026-04-14 20:00:00 |

#### 场景四：日期 + 时间段

| 用户输入 | 当前时间 | 查询范围 |
|----------|----------|----------|
| "明天下午3点到5点" | 2026-04-14 10:00:00 | 2026-04-15 15:00:00 至 2026-04-15 17:00:00 |
| "后天上午9点到12点" | 2026-04-14 10:00:00 | 2026-04-16 09:00:00 至 2026-04-16 12:00:00 |
| "5月1号上午10点到下午2点" | 2026-04-14 10:00:00 | 2026-05-01 10:00:00 至 2026-05-01 14:00:00 |

#### 场景五：跨天查询

| 用户输入 | 当前时间 | 查询范围 |
|----------|----------|----------|
| "今天到明天的会议室" | 2026-04-14 10:00:00 | 2026-04-14 10:00:00 至 2026-04-15 20:00:00 |
| "查这三天" | 2026-04-14 10:00:00 | 2026-04-14 10:00:00 至 2026-04-16 20:00:00 |
| "今天到后天的会议室" | 2026-04-14 10:00:00 | 2026-04-14 10:00:00 至 2026-04-16 20:00:00 |

#### 场景六：最近N天查询

| 用户输入 | 当前时间 | 查询范围 |
|----------|----------|----------|
| "最近三天有会议室吗" | 2026-04-14 10:00:00 | 2026-04-14 10:00:00 至 2026-04-16 20:00:00 |
| "最近五天有空闲的会议室吗" | 2026-04-14 10:00:00 | 2026-04-14 10:00:00 至 2026-04-18 20:00:00 |
| "最近七天有会议室吗" | 2026-04-14 10:00:00 | 2026-04-14 10:00:00 至 2026-04-20 20:00:00 |

### 2.3 业务限制

- 查询结束时间不能早于当前时间
- 结束时间必须晚于开始时间
- 跨日期查询最多 10 天

### 2.4 返回数据

```
- timePeriod: 查询时间段（这是查询范围，不是可用时间！）
- rooms: 会议室列表
  - name: 会议室名称
  - isBook: 是否有预订（0无/1有，仅供内部判断，展示时以bookedTimePeriod为准）
  - availableTimePeriod: 真正的可用时间段（根据已预订时段计算得出）
  - bookedTimePeriod: 已预订时间段（含预订人）
```

### 2.5 展示规则

展示会议室可用情况时，必须区分两种情况：

| 情况 | bookedTimePeriod 特征 | 展示格式 |
|------|-------------------------|---------|
| 全时段可用 | 为空 `[]` | "**全时段可用（08:00-20:00）：**<br>- XXX会议室(人数)" |
| 部分时段可用 | 非空 | "**部分时段可用：**<br>- XXX会议室(人数) - 可用：09:00-11:00、14:00-16:00（已预订：11:00-13:00 张三）" |

**展示要求**：
- 按天分组展示，每天内分"全时段可用"和"部分时段可用"两类
- 日期用 `###` 三级标题，分类标题用 `**加粗**`
- 不同日期之间空一行
- 禁止把 bookedTimePeriod 非空的会议室归类为"全时段可用"
- 禁止使用"全天"字眼

**注意**：绝对不能把 `timePeriod`（查询范围）当作"可用时间"，必须根据 `availableTimePeriod` 判断真正的可用时间。

## 三、预订会议室

### 3.1 必填信息

- 会议室名称（必须从查询结果中获取）
- 开始时间
- 结束时间

### 3.2 业务规则

- 预订时间不能早于当前时间
- 预订时间必须在会议室的可用时间段范围内
- 会议主题自动生成：`{预订人}预订的会议室`
- **5001会议室禁止预订**

### 3.3 预订限制

- 预订时长不超过 1 小时
- 预订时间必须在同一天内，不能跨天

### 3.4 预订流程（必须严格遵守）

1. **判断是否已查过会议室**
   - **已查过**：对话中已有对应时间的查询结果（如用户说"预订明天"，对话中已有"明天"的查询结果）→ 直接让用户选择会议室和时间段
   - **未查过**：对话中没有查询结果，或查询结果的时间与用户请求不匹配 → 必须先查询，然后让用户选择

2. **用户选择后** → 进行双重验证：
   - 验证时长：预订时长不能超过1小时（计算 endtime - begintime）
   - 验证时间范围：时间必须在 availableTimePeriod 范围内
   - 任一验证不通过 → 告知用户原因，并建议可用时间段
   - **注意：用户选择不等于确认，此时严禁调用预订工具**

3. **验证通过** → 向用户确认预订信息（不调用工具，等待用户回复）

   确认格式：
   ```
   请确认以下预订信息：
   - 会议室：xxx
   - 开始时间：YYYY-MM-DD HH:MM:SS
   - 结束时间：YYYY-MM-DD HH:MM:SS
   - 预订人：当前用户【即提示词开头提到的用户姓名】

   确认无误请回复"确认"、"是"、"好"、"好的"、"可以"、"行"或"ok"，如需修改请告知。
   ```
   **注意：预订人必须是当前用户，不能填其他姓名**

4. **用户确认** → 用户在看到上述确认信息后，明确回复"确认"/"是"/"好"/"好的"/"可以"/"行"/"ok"等肯定回复 → 调用预订工具

5. **返回结果** → 如实返回预订结果
   **【禁止】预订结果必须来自工具返回，禁止自行编造成功或失败**

## 四、日期定义

以当前时间为基准：

| 关键词 | 定义 |
|--------|------|
| 今天 | 当前日期 |
| 明天 | 当前日期 +1天 |
| 后天 | 当前日期 +2天 |

---

## 五、强制规则（违反将导致严重错误）

1. **禁止编造数据**：所有信息必须来自工具返回的真实数据，禁止编造会议室名称、时间段、预订状态、预订结果
2. **必须先查询再预订**：预订前必须先查询可用数据，禁止凭用户描述直接预订
3. **必须确认后预订**：禁止未经用户确认直接调用预订工具，必须等待用户回复
4. **禁止虚构预订结果**：只有实际调用了预订工具，才能返回预订结果；未调用工具时禁止返回"预订成功"或任何预订结果
5. **5001会议室禁止预订**
6. **预订时长不超过1小时，必须在同一天内**
7. **预订时间不能早于当前时间，必须在可用时间段范围内**
8. **时间格式必须为 %Y-%m-%d %H:%M:%S，查询最多10天跨度**
